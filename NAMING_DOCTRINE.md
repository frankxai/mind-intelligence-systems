## ✦ Naming Rules: Concept Level

### What "OS" Unlocks
When a repo bears the name OS, it signals:
- The user **lives inside it** daily
- It accumulates **personal canon** over time
- It is **opinionated** by design, not neutral infrastructure
- It has a **vault, journal, or context-store** component

### What "System" Unlocks
When a repo bears the name System, it signals:
- It is **installable** with defined inputs/outputs
- It solves a **specific domain problem** end-to-end
- It is **built for builders**, not just end-users
- It publishes **schemas, contracts, or packs** others can consume

### What "Systems" Unlocks
When a repo bears the name Systems, it signals:
- It is a **portfolio container** or governance layer
- It provides **cross-domain reusable primitives**
- It **maps relationships** between things
- It scales horizontally across many specializations
